# Engineering Playbook

Repositório responsável por centralizar os padrões, convenções e boas práticas utilizados nos projetos desenvolvidos por **Undertrone**.

O objetivo é garantir consistência entre diferentes projetos, facilitar a manutenção do código e reduzir decisões repetitivas durante o desenvolvimento.

## Objetivos

* Definir padrões de desenvolvimento.
* Padronizar a organização dos repositórios.
* Documentar convenções de Git e GitHub.
* Manter modelos reutilizáveis de documentação.
* Centralizar referências técnicas para projetos atuais e futuros.

## Documentação

### Git

Contém as convenções relacionadas ao controle de versão.

* Workflow de desenvolvimento
* Convenção de commits
* Convenção de branches
* Versionamento

### GitHub

Define padrões para organização dos repositórios.

* Estrutura dos repositórios
* Padrão de README
* Política de licenças
* Boas práticas de segurança

### Arquitetura

Padroniza a organização interna dos projetos.

* Estrutura de diretórios
* Convenções de nomenclatura
* Estilo de código
* ADR (Architecture Decision Records)

### Documentação

Define como a documentação técnica deve ser produzida e mantida.

### IA

Reúne diretrizes para utilização de ferramentas de inteligência artificial durante o desenvolvimento, incluindo organização de contexto, prompts e fluxo de trabalho.

## Templates

Contém arquivos reutilizáveis para acelerar a criação de novos projetos.

Exemplos:

* README
* CHANGELOG
* Issue Templates
* Pull Request Templates

## Reference

Material de consulta contendo exemplos de estruturas, organização de projetos e outras referências utilizadas como padrão.

## Princípios

Todos os projetos devem buscar:

* Consistência
* Clareza
* Simplicidade
* Escalabilidade
* Manutenibilidade
* Documentação adequada

## Evolução

Este repositório é evolutivo. Novos padrões, templates e convenções podem ser adicionados conforme surgirem novas necessidades ou tecnologias.
